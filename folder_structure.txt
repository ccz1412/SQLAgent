
# 项目目录结构说明

本项目按照以下组织方式进行代码、数据、实验与论文撰写管理。

```
项目根目录/
├── dat/          # 项目用到的数据集（原始或处理后数据）
├── vis/          # 数据/实验结果可视化（代码或生成图片）
├── doc/          # 论文撰写相关文件（pdf / doc / latex 源文件）
├── src/          # 创新模型的实现代码（核心算法/模型）
├── dep/          # 第三方依赖工程（子模块或外部库）
├── exp/          # 实验与中间结果
│   ├── csv/      # 表格形式实验结果
│   ├── results/  # 实验结果保存目录
│   └── eps/      # 矢量图格式结果（也可放于 vis）
├── api/script/   # 用于调用模型或数据的接口脚本
└── executable/   # 可执行文件或编译后的程序
```

## 目录详细说明

| 目录 | 用途 |
|------|------|
| `dat/` | 存放项目使用的数据集，包括原始数据或预处理后的数据文件。 |
| `vis/` | 可视化相关代码或生成的可视化图片（如 matplotlib 脚本、输出图片）。 |
| `doc/` | 论文写作相关文件，支持 pdf / doc / latex 格式。 |
| `src/` | 创新模型的核心实现代码，是本项目的主要代码目录。 |
| `dep/` | 第三方依赖工程，例如 Git 子模块或外部引用库。 |
| `exp/` | 实验与中间结果目录，内部可按需包含 `csv/`、`results/`、`eps/` 等子目录。 |
| `api/script/` | 提供调用模型或数据的接口脚本（如 REST API、数据处理脚本）。 |
| `executable/` | 存放编译后的可执行文件或可直接运行的二进制程序。 |

## 典型工作流程

1. **数据准备** → 放置数据文件到 `dat/`
2. **模型开发** → 在 `src/` 中实现核心模型，依赖 `dep/` 中的第三方库
3. **实验运行** → 执行实验，结果输出到 `exp/results/` 或 `exp/csv/`
4. **结果可视化** → 使用 `vis/` 中的脚本生成图表（如 `eps/`）
5. **论文撰写** → 在 `doc/` 中编写论文，引用 `vis/` 或 `exp/eps/` 中的图片

## 注意事项

- 请勿将大型数据集直接上传至代码仓库，可使用软链接或数据版本管理工具。
- `executable/` 中的二进制文件一般不建议纳入 Git 版本控制（视项目需求而定）。
- 实验中间结果（`exp/` 下）可根据需要选择性上传或忽略。
